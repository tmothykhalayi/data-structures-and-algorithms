## Practice Strategy

### Daily Practice Routine

#### Beginner Level (Weeks 1-4):
- **1-2 Easy problems** (30-45 min each)
- **Review solutions** (15-30 min)
- **Note patterns** (10 min)

#### Intermediate Level (Weeks 5-12):
- **1 Easy + 1 Medium** (1.5-2 hours total)
- **Or 2 Medium problems** (2-2.5 hours total)
- **Review and optimize** (30 min)

#### Advanced Level (Weeks 13+):
- **1 Medium + 1 Hard** (2.5-3 hours)
- **Or 2-3 Medium problems** (2.5-3 hours)
- **Pattern documentation** (20 min)

---

## Problem Selection Strategy

### Phase 1: Topic-Based (Weeks 1-8)
Focus on one topic at a time:
- Master the fundamentals
- Solve 15-20 problems per topic
- Move to next topic only after confidence

### Phase 2: Pattern-Based (Weeks 9-12)
Focus on patterns across topics:
- Two pointers
- Sliding window
- Fast/slow pointers
- DFS/BFS
- Backtracking
- DP patterns

### Phase 3: Mixed Practice (Weeks 13-16)
Random problem selection:
- Simulates real interviews
- Improves pattern recognition
- Builds versatility

## Problem-Solving Checklist

**Before coding**:
- [ ] Understand the problem fully
- [ ] Ask clarifying questions
- [ ] Work through examples
- [ ] Identify pattern/approach
- [ ] Discuss brute force
- [ ] Optimize approach
- [ ] Estimate time/space complexity

**During coding**:
- [ ] Think out loud
- [ ] Write clean code
- [ ] Use meaningful variable names
- [ ] Handle edge cases
- [ ] Test as you code

**After coding**:
- [ ] Walk through code
- [ ] Test with examples
- [ ] Check edge cases
- [ ] Optimize if needed
- [ ] Analyze complexity

## Final Tips

### For Success:
- **Consistency > Intensity**: Daily practice beats cramming
- **Quality > Quantity**: Understand deeply over solving many
- **Review > New**: Reviewing old problems is valuable
- **Explain > Memorize**: Understanding beats rote learning
- **Struggle > Give up**: Struggling builds skill

### Interview Day:
- Get good sleep
- Review common patterns (not new problems)
- Stay calm and confident
- Think out loud
